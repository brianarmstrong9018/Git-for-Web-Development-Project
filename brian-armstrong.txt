    1. What is Semantic HTML? 
    It's HTML that gives meaning to our markup
    2. What is HTML used for? 
    It's used for the structure of our website
    3. What is an attribute and where do we put it?
    HTML attributes provide additional information about HTML elements. They are put after the element. 
    4. What is the h1 tag used for? How many times should I use it on a page?
    It is used for the main heading or title of the page. It should only be used once.
    5. Name two tags that have required attributes
    <a> and <img>
    6. What do we put in the head of our HTML document? 
    Meta data about the webpage.
    7. What is an id? 
    It uses the id attribute of an HTML element to select a specific element.
    8. What elements can I add an id to? 
    Any HTML elements in the body of the webpage. 
    9. How many times can I use the same id on a page? 
    You cannot have more than one element with the same id in an HTML document.
    10. What is a class? 
    A group of elements that are the same or similar.
    11. What elements can I add a class to? 
    Any HTML elements in the body of the webpage. 
    12. How many times can I use the same class on a page? 
    There is no limit.
    13. How do I get my link to open in a new tab?
    Add target="_blank" to the anchor tag.
    14. What is the alt attribute in the image tag used for? 
    It provides alternate text for an image, if the image cannot be displayed.
    15. How do I reference an id?
    Write the ID with a "#" before it.
    16. What is the difference between a section and a div
    Section indicates that the data within relates to a single theme. Div is more generic.
    17. What is CSS used for? 
    The styling of a webpage.
    18. How to we select an element? Example - every h2 on the page
    .h2 will select every h2 on the page.
    19. What is the difference between a class and an id? - Give me an example of when I might use each one
    Class will select all elements that are specified to have that class. ID will select just a single HTML element.
    20. How do we select classes in CSS?
    Write ".classname"
    21. How do we select a p element with a single class of “human””?
    p.human
    22. What is a parent child selector? When would this be useful? 
    It selects all elements that are a direct child of the specified element. When you need to target a specific group of elements.
    23. How do you select all links within a div with the class of sidebar?
    div > sidebar
    24. What is a pseudo selector?
    It is a keyword added to a selector that lets you style a specific part of the selected element.
    25. What do we use the change the spacing between lines?
    line-height
    26. What do we use to change the spacing between letters?
    letter-spacing
    27. What do we use to to change everything to CAPITALS? lowercase? Capitalize?
    uppercase, lowercase, capitalize
    28. How do I add a 1px border around my div that is dotted and black?
    div {
        border:1px dotted black;
    }
    29. How do I select everything on the page? 
    *
    30. How do I write a comment in CSS?
    /* This is a CSS comment */
    31. How do I find out what file I am in, when I am using the command line? 
    pwd
    32. Using the command line - how do I see a list of files/folders in my current folder?
    ls
    33. How do I remove a file via the command line? Why do I have to be careful with this? 
    rm. It will delete the file permanently.
    34. Why should I use version control? 
    To keep track of changes made.
    35. How often should I commit to github?
    Every time there has been a major change / or every 20 minutes.
    36. What is the command we would use to push our repo up to github? 
    git push
    37. Walk me through Lambda's git flow. 
    First you will need to fork the repo. Next, select "Clone or Download" and copy the URL, then cd to the directory you want to download to. Next, run "git clone linktorepo." Then cd
    into the new repo directory and run "git checkout -b 'firstname-lastname'" to create your own branch. Next, work on repo and make changes. Then run "git add ." to addrepo changes and make them ready for committing.
    Next, run git commit -m "Your helpful commit message here". Then, git push -u origin branch-name will push all of your commits from your local machine up onto github. Now, navigate back to
    the forked page on Github. Select Branch:master button on the left side. This will create a dropdown where you should be able to select your name-branch. Click the dropdown button that says base repository and 
    and change it to be your forked repo. Next, select create pull request.
